Payment fraud is a major risk for e-commerce platforms because it can lead to financial loss, chargeback costs, and reduced customer trust, while overly strict fraud controls may incorrectly block legitimate customers and create unnecessary friction. This report presents an end-to-end business analytics project for real-time payment fraud detection and risk scoring using the Online Payments Fraud Detection dataset, extended with synthetically generated contextual and behavioral features such as account age, device signals, failed payment attempts, address mismatch, and time-based transaction patterns. The project follows the full analytics lifecycle, including business problem definition, data generation, exploratory data analysis, data cleaning, feature engineering, model development, deployment, and monitoring. Since fraud data is highly imbalanced, the models are evaluated using business-relevant metrics such as precision, recall, F1-score, AUC-PR, and cost-based analysis rather than accuracy alone. The selected model is deployed as a real-time scoring service that assigns each transaction a fraud probability, risk level, and recommended action, such as auto-approval, manual review, or blocking. The project also includes monitoring components to track data drift, prediction drift, and rolling model performance, demonstrating how machine learning and streaming analytics can support fraud loss reduction while maintaining a smooth customer payment experience.
